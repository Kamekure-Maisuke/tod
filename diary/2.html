使用技術及び仕様

このホームページを作成するにあたっての技術選択及び仕様としては以下。

  • コンテンツ : HTML
    • 理由 : 長年使われている標準マークアップ言語であり、今後も使われていくだろうから。
  • バージョン及びコンテンツ管理 : GitHub
    • 理由 : バージョン管理のデファクトスタンダードであり、北極で1000年間保存してくれるから。
    • 参照
  • アプリケーション構成 : MPA
    • 理由 : 現代において表示速度の差が気になる事はほぼないから。コンテンツサービスとして不要な技術を取り込みたくないから。
  • ホスティング : Vercel
    • 理由 : Githubとの連携が簡単だから。ホスティングサービスの中で一番楽ができるから。
  • 日記の投稿日時 : 無し
    • 理由1 : 「投稿日時無し」の日記でも読まれる文章を書けるか残したくなったから。
    • 理由2 : 日時自体はバージョン管理サービスで判断できるから。
  • フレームワーク : 無し
    • 理由 : 依存したくないから。
  • CSS及びJS : 無し
    • 理由 : 単一HTMLで済むから。

上記の内容にした理由は、長く続けられる方法を模索していたからである。

私は特性に加えて不摂生な生活によって、人一倍自身の身体を上手に扱うことができないのである。

年々脳や身体の機能が衰えてきている私でも作成を続けられる仕組みにしたかったのである。

ただ、サーバー管理やコンテンツ管理はできるだけ最新のものを使って楽をしたいという思いもある。

つまり、「低依存性・長期性・楽さ」をサービスや標準団体に乗っかって実現して、私はちまちまと飲みながら日々投稿を続けたいのである。
